SQL

SQL注入是一种将SQL代码插入或添加到应用（用户）的输入参数中，再将这些参数传递给后台的SQL服务器加以解析并执行。

如果 web 应用未对动态构造的SQL 语句所使用的参数进行正确性审查，那么攻击者就很可能修改后台 SQL 语句的构造。

如果攻击者能够修改 SQL 语句，那么该语句将与应用的用户拥有相同的运行权限。

存在漏洞的位置

1）动态网页

PHP，ASP等，由相应的脚本引擎来解释执行，根据指令生成静态网页。

特点：

1） 依赖数据库

2） 灵活性好

3） 交互性好，功能强大

4） 存在安全风险，可能存在SQL注入漏洞

2）伪静态网页

“伪静态”顾名思义就是一种表面上看似是静态网页（以.html、.htm等结尾），不存在任何的数据交互，却其实是动态网页，存在数据交互的网站，具有这种特性的网页被称为“伪静态网页”。我们看到的伪静态网页其实是经过处理的，将动态网页的id等参数通过URL重写来隐藏，让查看者以为是静态网页

特点：在平时的测试过程中我们经常会看到这样一类奇特的地址：http://xxx.xxx.xx.xx:xxx/test.php/id/1’.html （原型一般为：http://xxx.xxx.xx.xx:xxx/test.php?id=1’ ）。像这种类型的网址URL，往往是一种伪静态网页，遇到这种情况可以直接对”.html”之前的参数加“’”进行判断是否存在注入！

攻击手段

  1. 判断应用程序是否存在注入漏洞

  2. 收集信息、并判断数据库类型mysql

  3. 根据注入参数类型，重构SQL语句的原貌

  4. 猜解当前库名、表名、字段名，查询数据

  5. 获取账户信息、攻击web或为下一步攻击做准备

危害

1） 数据库信息泄露

2） 网页篡改

3） 数据库被恶意操作

4） 服务器被远程控制

5） 种植木马

类型

根据注入点类型划分：

1） 数字型注入点
形式：http://xxx.com/news.php?id=1

2） 字符型注入点
形式：http://xxx.com/news.php?name=admin

3） 搜索型注入点
形式：http://xxx.com/news.php?keyword=admin

根据数据提交方式划分：

1）GET注入

提交数据的方式是 GET , 注入点的位置在 GET 参数部分。

2） POST注入

使用 POST 方式提交数据，注入点位置在 POST 数据部分，常发生在表单中。

3） HTTP头部注入

HTTP 请求的时候会带上客户端的 Cookie, 注入点存在 Cookie 当中的某个字段。

4） Cookie注入

注入点在 HTTP 请求头部的某个字段中，比如存在 User-Agent 字段中。

按照执行效果划分：

1） 联合查询注入

可以使用union的情况下注入。

2） 报错型注入

即页面会返回错误信息，或者把注入的语句结果直接返回在页面中。

3） 基于时间的盲注

即不能根据页面返回内容判断任何信息，用条件语句查看时间延迟语句是否执行（即页面返回时间是否增加）进行判断。If() ,Sleep(5)

4） 基于布尔的盲注

即可以根据返回页面判断条件真假的注入

5） 堆查询注入

5）可以同时执行多条SQL语句的注入。

联合查询注入 union select

union操作符用于合并两个或多个select语句的结果集。

union内部的select语句必须拥有相同数量的列，列也必须拥有相似的数据类型。同时，每条select语句中列的顺序必须相同。

Order by语句

用于根据指定的列对结果集进行排序。（判断列数）

默认按照升序对记录进行排序。

单引号有个特殊的作用：命令分隔符

步骤

  1. 求闭合字符

  2. 求列数

  3. 求显示位

  4. 爆数据库名

  5. 爆表名

  6. 爆列名

  7. 爆字段名

布尔盲注

当某一页面存在注入，但是没有显示位，而且没有数据库报错信息，只能通过页面返回正常和不正常进行判断，那么需要布尔型注入。

应用程序仅仅返回True（页面）和False（页面），可以通过构造逻辑判断（比较大小）来得到我们需要的信息。

优点：

1）不需要显示位

2）不需要报错输出mysql_error()信息

缺点：速度慢

步骤

  1. 求闭合字符

  2. 求当前数据库名的长度

  3. 求当前数据库名对应的ascii值

  4. 求表的数量count

  5. 求表名的长度length

  6. 求表名对应的ascii值

  7. 求列的数量

  8. 求列名的长度

  9. 求列名对应的ascii值

  10. 求字段的数量

  11. 求字段内容的长度

  12. 求字段内容对应的ascii值

布尔盲注

当某一页面存在注入，但是没有显示位，而且没有数据库报错信息，只能通过页面返回正常和不正常进行判断，那么需要布尔型注入。

应用程序仅仅返回True（页面）和False（页面），可以通过构造逻辑判断（比较大小）来得到我们需要的信息。

优点：

1）不需要显示位

2）不需要报错输出mysql_error()信息

缺点：速度慢

步骤

  1. 求闭合字符

  2. 求当前数据库名的长度

  3. 求当前数据库名对应的ascii值

  4. 求表的数量count

  5. 求表名的长度length

  6. 求表名对应的ascii值

  7. 求列的数量

  8. 求列名的长度

  9. 求列名对应的ascii值

  10. 求字段的数量

  11. 求字段内容的长度

  12. 求字段内容对应的ascii值
